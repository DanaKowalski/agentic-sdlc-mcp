# Environment Setup

Follow these steps before writing any code. Do not begin implementation until the verification command in step 6 succeeds.

---

## Prerequisites

1. Confirm Node.js version is 18 or higher:
   ```bash
   node --version
   ```
   Expected output: `v18.x.x` or higher. If not, install Node 18+ before continuing.

2. Confirm npm is available:
   ```bash
   npm --version
   ```
   Any version bundled with Node 18+ is acceptable.

3. No global tools are required beyond Node and npm. Do not install global packages to work around a missing dependency — add it to `package.json` instead.

---

## Initial Setup

4. Clone the repository (skip if already cloned):
   ```bash
   git clone <repository-url>
   cd <project-directory>
   ```

5. Install dependencies:
   ```bash
   npm install
   ```
   If this fails, check that Node 18+ is active and that you are in the repository root (the directory containing `package.json`).

6. Build the MCP server:
   ```bash
   npm run build
   ```
   Expected outcome: no TypeScript errors, the project's build output directory is created. If TypeScript errors appear, do not proceed — fix or report them before writing new code.

---

## Environment Variables

There is no `.env.example` file in this repository at this time. Environment variables are only required for tools that call external APIs (e.g. GitHub token for issue search). If your task introduces a new environment variable:

- Add it to `.env.example` (create the file if it does not exist yet)
- Document it with a comment explaining what it is used for and whether it is required for local development, tests, or both
- Never commit actual values — `.env` is gitignored

If your task does not call external services, no environment variables are needed and this section does not apply.

---

## Verifying the Setup

7. Run the type-check to confirm the build is clean:
   ```bash
   npm run build
   ```
   A clean build produces output ending in:
   ```
   Found 0 errors.
   ```
   or exits with code 0 with no error output.

8. If the project provides a config or state validation script, run it now and confirm it passes. Check the project's `package.json` scripts for the relevant command.
   This must pass before and after your changes. If it fails before you have made any changes, report it to the orchestrator — do not continue.

---

## Starting the MCP Server for Manual Testing

9. After a successful build, start the server using the project's server start command (defined in the project's `package.json`):
   ```bash
   [your project's server start command]
   ```
   The server reads from `stdin` and writes to `stdout` using the MCP protocol. To test manually, connect an MCP client (such as Claude Desktop or a local MCP inspector) to the process.

10. To rebuild and restart after making changes:
    ```bash
    npm run build && [your project's server start command]
    ```

---

## Common Setup Failures and Fixes

**Problem:** `npm install` fails with a dependency conflict or resolution error.  
**Fix:** Confirm you are on the correct Node version for this project. Check the project's README or `engines` field in `package.json` for the required version. Retry after switching to the correct version.

**Problem:** The build fails with a missing module error.  
**Fix:** Confirm `npm install` completed successfully and all dependencies are present in `node_modules/`. Run `npm install` and retry the build.

**Problem:** The build fails with TypeScript type errors in files you did not modify.  
**Fix:** The codebase may have pre-existing errors, or a dependency type definition changed. Report this to the orchestrator before proceeding — do not mask the errors to make the build pass.

**Problem:** A config or state check script fails on a fresh clone.  
**Fix:** The script may require a baseline file generated by a companion script. Read the script's error output to identify the missing file, then run the generator script the project provides for that purpose.

**Problem:** The server process exits immediately after starting.  
**Fix:** The build output is missing or stale. Run the build command and confirm it succeeds with no errors before starting the server again.
